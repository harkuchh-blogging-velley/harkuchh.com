Strong Passwords, Weak Passwords: How to Generate Strongest Possible Passwords Which You Can Remember

Feb 4th, 2014

Passwords we know, the most common method used by almost all websites for authenticating users. But how good is this method? Well, the strength of this method partially depend on the strength of the password we choose.

What’s the strength of a password?

Strength of a password is determined by a number of things.

  • Character set
    Most important thing which make a password strong is its character set. Character set of a password means which characters are used in the password, like lower case characters, upper case characters, numbers, symbols etc. A password which has all different type of characters is considered stronger.
  • Length of the password
    Second most important thing about strong passwords is their length. Longer passwords are considered more secure.

  • Guessability of the password
    Another thing which determine strength of password is of easy it is to guess a password. Easier to guess passwords are weak. A security researcher has said that any password you can remember is not safe enough.

    The problem

    So we have a problem here. They say any password we choose which we can remember is not safe enough. Most people choose easy, simple to remember password, which is perfectly reasonable. Why putting extra load on brain? But this laziness can cost more than you think.

    The Solution


    So I wanted a system which will create passwords for me which I can remember, but which can’t be guessed or cracked easily. One solution to this is passphrases. For example, instead of choosing a single word password like “facebook123”, I choose a sentence like “facebookhasbluecolor”. And even better, change some character with symbols and numbers, “f@c3b00kh4sBlu3C0lOr”. Awesome, right! But it still can be remembered. So according to our theory that any password which can be remembered is not safe enough. I want something simple to remember and type and still very strong and long. So being a software developer, I made a system which do exactly this. I call it fireword.

    What is fireword?

    Fireword is an application in which you enter a simple word or phrase, and it give you a very strong and impossible to guess password.

  • Fireword always generate same password for same word. So you remember your simple password, and generate a strong one with this site. And then when you need to login again, regenerate strong password with your simple one.

  • Fireword allow you to select the length of the password. You can choose from 8 to 128 character long password.
  • For example, just entering ‘facebook’ become this ‘bF2A37d5e8%{61?C]>3E[{(9{’. Any password can be expanded to at most 128 character long complex password.

    Recommended ways to use Fireword

    It’s really simple. Instead of using you easy to remember but weak password for a website, change your password with the password generated by fireword.

  • Enter your simple password in Fireword, get a strong one, and use it for logging in.
  • Do not copy-paste strong generated passwords on your hard-disk, it is very insecure. Instead, whenever you need to login again, just get back to the fireword, and generate password again. Fireword will always generate same strong password for same simple password you enter.

    How to choose passwords?

  • Instead of choosing ridiculously stupid passwords like ‘password’, you should choose little smarter passwords. Even if you are hardening your passwords with fireword, try choosing a better password.
  • Choose different passwords for different sites following some pattern, like
  • Use the name of the website for which you are generating password and use some words in front and end of it. Like For facebook choose bluefacebookaccount For google choose greengoogleaccount Come up with your own patterns. You can even use your phone number, your name and the site name in the combo. Convert the pattern with fireword, and get strongest password possible. Amazing. Isn’t it!
