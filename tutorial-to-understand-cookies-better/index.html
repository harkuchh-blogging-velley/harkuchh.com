Tutorial to Understand 'COOKIES' Better

In our post ”What are cookies and how to use them in PHP ?” we said in the last of  post that ‘Nothing teaches better than a quick tutorial’. So here’s one. I hope you have already set a php development environment. If not, learn how to do it on Linux here.   We’ll make three files in this tutorial.
  1. login.php: Page with login form
  2. login_proc.php: Page with code to set cookies on successful login
  3. user.php: User profile page that’ll use information set in cookies
 

login.php

<table>

<form action=”login_proc.php” method=’post’>

<tr>

<td>

  <lable for=’user_name’>Enter your Name</lable>

</td>

<td>

  <input type=”text” name=’user_name’>

</td>

</tr>

<tr>

<td>Password</td>

<td>

  <input type=”password” name=’passwd’>

</td>

</tr>

<tr>

<td>

  <input type=”submit” value=’Log In’>

</td>

</tr>

</form>

</table>

  Nothing to explain in this file. Just a simple form  

login_proc.php

 

<?php

if($_POST[‘passwd’] == ‘root’){

setcookie(’name’,$_POST[‘user_name’], time()+3600);

header(”location: user.php”);

}

else{

header(”Location: login.php”);

}

?>

  This code check if password was right (here you put your login validation logic) and sets a cookie with name ‘user_name’. In real world projects, here come the user_id or unique user_name (actually in an encrypted form so that user don’t access other users’ accounts by changing cookie values().  

user.php

 

<?php

echo “<h1>Hello “.$_COOKIE[‘name’].”</h1>”;

?>

  Here is the code of user’s profile page. Not much in this example. But in real world, use cookie variable to identify the user and show him a custom profile page (with his personal information).   Above tutorial was just an illustration of how to deal with cookies.
